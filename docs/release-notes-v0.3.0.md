# ganttly v0.3.0 — 多项目工作区

> 从单项目编辑器升级为本地多项目工作区。一次打开、切换、管理你所有的甘特计划。

🎨 **在线 demo**:https://chiangzg.github.io/ganttly/

---

## 主要特性

### 多项目工作区

把 ganttly 从「一个项目一个窗口」升级为「一个工作区管所有项目」。

| 功能                   | 说明                                                                   |
| ---------------------- | ---------------------------------------------------------------------- |
| **项目中心**           | 统一的项目列表入口,展示所有本地项目,支持搜索、筛选                     |
| **项目级路由**         | 每个项目拥有独立 URL(`/project/<id>`),可直接分享、收藏、刷新不丢上下文 |
| **顶部快速切换**       | 标签页(Tabs)形态,已打开的项目一键切换,关闭即回到中心                   |
| **收藏 / 固定**        | 常用项目收藏置顶,避免在长列表里翻找                                    |
| **复制项目**           | 一键复制现有项目作为模板,保留任务结构与依赖                            |
| **回收站**             | 删除进回收站,支持恢复,防误删                                           |
| **项目级保存生命周期** | 自动保存按项目隔离,切换项目互不干扰                                    |

### 数据层升级

- **修订感知仓储(revision-aware repositories)**:读写带版本校验,并发切换不串数据。
- **IndexedDB / LocalStorage v2 迁移**:从 v0.1/v0.2 的单项目存储平滑迁移到多项目结构,旧数据自动升级,`default` 项目 ID 保留。
- **导入即新建项目**:导入 JSON / `.gan` 现在会创建一个新项目,而非覆盖当前;导出仍按当前活动项目作用域。

## 配套修复

- **滚动错位修复**:任务视图与资源视图左右滚动时,任务条、依赖箭头、网格线与表头跟随 `scrollTop` 同步,不再错位。([`f25400e`](https://github.com/chiangzg/ganttly/commit/f25400e))
- **性能测试稳定化**:修复 CI 上 `1000-task` FPS 测试的 flaky 失败 —— 阈值改为 CI 感知(CI 15 / 本地 30),采样改为走真实 store 路径而非合成事件。([`98472a9`](https://github.com/chiangzg/ganttly/commit/98472a9))

## 不变项

- `GanttlyFile` 数据 schema **保持不变**,旧版本数据可直接打开。
- 服务端 API 与实时协同仍不在本期范围。
